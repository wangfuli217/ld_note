控制台：main
Win32窗口程序：WinMain
动态库DLL：DllMain

编译链接后存在的形式不同
控制台和Win32窗口程序：EXE文件，可以执行
动态库DLL：DLL文件，无法执行
静态库LIB：LIB文件，无法执行

2.1 开发和编译环境
VC1.5 - VC6.0(2008) - VC10.0(2010)  
2.1.1 编译器 CL.EXE
  将原代码编译目标代码. OBJ文件.
2.1.2 链接器 Link.EXE
  将OBJ文件和库函数链接.生成EXE.
2.1.3 资源编译器 rc.exe
  将资源脚本编译成可以链接的文件(RES).
  
OutputDebugString
<windows.h> 文件声明了 OutputDebugString() 函数的两个版本 - 一个用于 ASCII，一个用于 Unicode - 
不像绝大多数 Win32 API 一样，原始版本是 ASCII。而大多数的 Win32 API 的原始版本是 Unicode。
使用一个 NULL 结尾的字符串缓冲区简单调用 OutputDebugString() 将导致信息出现在调试器中，如果有调试器的话。
